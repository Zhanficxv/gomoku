游戏注册体系

一个主程序，多个小游戏，既能统一接入，也能单独部署。

每个小游戏都拥有自己的独立目录、入口页面与静态资源，主程序通过注册表统一展示与挂载。 后续继续新增小游戏时，只需要新增目录并注册元数据，即可自动出现在大厅中。

扩展约定

注册后自动接入大厅

主程序通过 /api/arcade/games 提供游戏清单。大厅会自动渲染入口卡片、标签、玩法特性与独立部署路径。

  • 独立目录：/web/static/games/<slug>/
  • 独立入口：每个游戏都有自己的 index.html
  • 统一注册：主程序集中管理路由展示信息
  • 后续扩展：新增目录 + 注册即可

前端目录与部署方式

  • /web/static/index.html：统一大厅首页
  • /web/static/games/<slug>/：每个小游戏的独立静态资源目录
  • 主程序通过注册表统一展示全部小游戏
  • 单个小游戏目录也可以被单独拷贝并作为静态站点部署
