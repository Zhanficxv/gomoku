# Arcade Hub · 小游戏合集

一个基于 Go + 原生前端实现的小游戏合集项目，目前共 **6 款游戏**：

| 游戏 | 类型 | 主要特性 |
| --- | --- | --- |
| **五子棋（Gomoku）** | 策略对弈 · 含账号 | 登录 / 注册 / 会话恢复 / 专属对局隔离 |
| **连连看（Link Up）** | 休闲消除 · 纯前端 | 6×6 图块、两次转弯连通、计时与步数 |
| **俄罗斯方块（Tetris）** | 街机 · 落体堆叠 | SRS 标准旋转、7-Bag 随机、持有、幽灵预览、硬降、锁定延迟、计分与等级 |
| **扫雷（Minesweeper）** | 逻辑推理 · 经典 | 初/中/高/自定义 四档难度、首击安全、插旗、双击翻开 |
| **打砖块（Breakout）** | 街机 · 反弹 | 5 个手工关卡 + 7 种掉落道具（多球 / 桨加宽 / 慢球 / 激光 / 加命 / 桨变窄 / 快球） |
| **井字棋（Tic-Tac-Toe）** | 双人 / 人机 | 三档 AI（随机 / 启发式 / 完美 Minimax），完美难度下永不输 |

技术栈：

- **后端**：Go 1.22，使用标准库 `net/http` 提供账号与五子棋 API
- **前端**：原生 HTML + CSS + JavaScript，无框架、无构建步骤
- **部署**：单二进制运行，前端静态资源通过 `embed` 嵌入

## 当前功能

### 游戏大厅

- 根目录为统一小游戏大厅，列出所有可玩游戏
- 记录最近一次进入的小游戏入口
- 支持继续进入上次游玩的游戏
- 前端目录按子游戏拆分，便于后续继续扩展

### 五子棋

- 15×15 标准棋盘，黑棋先行
- 五子（含长连）连珠即胜，自动判定平局
- 高亮最后一手与获胜连线
- 鼠标悬停预览落子位置
- 悔棋 / 重置棋盘 / 新对局
- 落子记录列表
- 登录 / 注册 / 退出登录
- 基于 Cookie 的会话保持，刷新页面后自动恢复登录态
- 用户间对局隔离，每位用户仅能访问自己的游戏

### 连连看

- 6×6 图块棋盘
- 相同图案且路径拐点不超过两次时可消除
- 支持重新开始与重新洗牌
- 显示剩余图块、已完成配对、步数与计时

### 俄罗斯方块

- 标准 SRS（Super Rotation System）旋转 + 完整 wall-kick 表（含 I 系单独表）
- 7-Bag 随机器：每袋包含 7 种方块各一次，避免长时间出现同色或缺色
- **持有方块**（Hold）：每次落地前可触发一次切换
- 幽灵预览（Ghost piece）显示当前方块若直接下落的落点
- 硬降（Space）+ 软降（↓）+ 锁定延迟（最多 15 次重置）
- 计分：单/双/三/Tetris + Combo + 软降/硬降奖励
- 等级随消行自动提升，重力按指南曲线递增
- 桌面键盘 + 手机底部触摸操作栏 + 棋盘滑动手势

### 扫雷

- 四档难度：初级 9×9 / 10 雷、中级 16×16 / 40 雷、高级 30×16 / 99 雷、自定义
- 首次点击保证安全（在 3×3 安全区外才放置地雷）
- 左键揭开 / 右键插旗 / 长按插旗（移动端）
- 数字格 **双击** 自动展开周围（chord）
- 自动泛洪展开 0 邻居区域
- 计时器、雷计数、表情按钮、本地最佳时间记录

### 打砖块

- 5 个手工设计的关卡，难度递增
- 砖块类型：普通 1 HP / 强韧 2 HP / 重型 3 HP / 蓝色高分 / 不可破坏
- 7 种掉落道具：
  - 多球 ×3（球分裂）
  - 桨加宽（陷阱：桨变窄）
  - 慢球（陷阱：快球）
  - 激光武器（按 Space 自动开火）
  - 加 1 命
- 鼠标 / 触屏 / 键盘三种控制
- 桨上反弹位置决定球出射角度（街机风格）
- 本地最佳分数

### 井字棋

- 双人对战 + 人机对战 两种模式
- 人机三档难度：
  - 简单 — 纯随机
  - 中等 — 启发式（赢手 / 堵手 / 中心 / 角落优先），约 25% 偶发失误
  - 困难 — 完美 Minimax，最佳结局为平局
- 可选玩家执子（X 先手 / O 后手）
- 比分与回合计数

## 快速开始

### 1. 运行

需要 Go 1.22+：

```bash
go run .
```

默认监听 `:8080`，浏览器打开 [http://localhost:8080](http://localhost:8080) 即可进入小游戏大厅。

### 2. 构建单二进制

```bash
go build -o arcade-hub .
./arcade-hub -addr :8080
```

可使用环境变量 `GOMOKU_ADDR` 或 `PORT` 配置监听地址。

### 3. 运行测试

```bash
go test ./...
```

## 路由说明

### 页面路由

- `/`：小游戏大厅
- `/games/gomoku/`：五子棋
- `/games/linkup/`：连连看
- `/games/tetris/`：俄罗斯方块
- `/games/minesweeper/`：扫雷
- `/games/breakout/`：打砖块
- `/games/tictactoe/`：井字棋

### HTTP API

所有响应均为 JSON。

| 方法 | 路径                   | 说明                       |
| ---- | ---------------------- | -------------------------- |
| POST | `/api/auth/register`   | 注册用户并自动登录         |
| POST | `/api/auth/login`      | 用户登录                   |
| POST | `/api/auth/logout`     | 退出登录                   |
| GET  | `/api/auth/me`         | 获取当前登录用户           |
| POST | `/api/games`           | 创建一局新的五子棋对局     |
| GET  | `/api/games/{id}`      | 获取五子棋对局状态         |
| POST | `/api/games/{id}/move` | 五子棋落子                 |
| POST | `/api/games/{id}/undo` | 五子棋悔最后一步           |
| POST | `/api/games/{id}/reset` | 重置五子棋棋盘            |
| GET  | `/healthz`             | 健康检查                   |

> 除 `/api/auth/*` 与 `/healthz` 外，其余接口都需要先登录，并且只能访问当前用户自己的五子棋对局。

新增的俄罗斯方块、扫雷、打砖块与井字棋全部为纯前端实现，没有后端 API，进入游戏页面即可游玩。

## 技术说明

- 五子棋游戏逻辑使用读写锁保护，所有对局可并发访问
- 用户信息与会话默认保存在内存中，适合示例项目与单机部署
- 连连看 / 俄罗斯方块 / 扫雷 / 打砖块 / 井字棋 均为纯前端实现，不依赖后端状态
- 静态资源通过 `embed.FS` 打包，部署只需单个二进制
- 所有页面均做了响应式适配，桌面、平板与手机均可游玩
